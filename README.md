# Botamin · Дашборд аналитика

Инструмент, который за минуту показывает аналитику: как идут дела на проекте,
**на каком шаге бот теряет клиента**, и куда направить усилия для роста конверсии.

## Запуск

```bash
python3 -m venv .venv
.venv/bin/pip install streamlit pandas plotly
.venv/bin/streamlit run app.py
```

Открыть http://localhost:8501

Путь к данным задаётся в `app.py` → `DATA_PATH` (по умолчанию `Calls Week Anon.csv`).

## Состав

| Файл | Что делает |
|---|---|
| `app.py` | Streamlit-дашборд (воронка, разбор утечки, время, качество, drill-down) |
| `data_pipeline.py` | Загрузка + разметка: шаги воронки, причины обрыва, реакции, баги качества, перезвоны |
| `РАЗБОР.md` | Сопровождение: какие метрики и почему, что в данных, точка роста и первый A/B |

## Как читать дашборд

1. **KPI + воронка** — общая картина и где главный обрыв.
2. **Где теряем на шаге 1** — разбор главной утечки на типы (молчание / сброс / отказ).
3. **Куда направить усилия** — переходы воронки по взвешенной потере контактов.
4. **Реакции клиентов** — что чинить в скрипте (по вовлечённому хвосту).
5. **Когда звонить** — вовлечённость по часам и дням.
6. **Качество бота** — готовые дефекты (баг отрасли, зацикливание).
7. **🔎 Разбор диалогов** — фильтруешь срез и читаешь реальные транскрипты обрывов.

## Методология разметки (коротко)

Скрипт бота шаблонный, поэтому **шаг, до которого дошёл разговор**, определяется
детерминированно по маркерам в репликах бота (приветствие → оффер → встреча →
квалификация). Причина обрыва на шаге 1 и тип реакции клиента — эвристики по
тексту. LLM в пайплайне не используется (всё воспроизводимо), но предусмотрен
хук: если положить рядом `llm_labels.csv` с колонками `phone, dt_raw, <метки>`,
они подмешаются автоматически — так разметку реакций можно усилить через Claude.
